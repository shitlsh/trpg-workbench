# Skill 使用指南

Skill 是一种持久化的 Agent 创作框架指令。当你在工作区中添加 Skill 后，AI Agent 在执行 Workflow（创建模组、修改资产等）时会自动读取并应用它，无需每次在对话中重复说明。

---

## 什么是 Skill？

Skill 以 Markdown 文件形式存储在工作区数据目录中（`skills/` 子目录），每个 Skill 包含：

- **名称**：Skill 的标识
- **描述**：一句话说明这个 Skill 的作用（Director 可据此感知工作区状态）
- **适用 Agent 类型**：指定哪些 Agent 会应用这个 Skill（留空 = 所有创作型 Agent）
- **内容**：注入到 Agent 的具体框架指令（Markdown 格式）
- **启用开关**：禁用后不会注入，但文件保留

---

## Skill 能做什么？

Skill 的典型用途是**为特定类型的资产定义结构化输出要求**。例如：

- 要求 NPC Agent 在每次创作时必须包含"神话接触程度"和"心理稳定性"字段
- 要求怪物 Agent 区分物理威胁和精神威胁两个维度
- 要求所有 Agent 在创作时考虑某个特定的世界观背景

---

## Skill 文件格式

Skill 文件使用 YAML frontmatter + Markdown 正文：

```markdown
---
name: CoC NPC 框架
description: 为克苏鲁神话体系 NPC 提供调查员维度结构
agent_types: [npc]
enabled: true
---

在创作 NPC 时，必须包含以下维度：
- 职业（1920s 社会角色）
- 神话接触程度：无 / 轻微 / 深度
- 心理稳定性：正常 / 不稳定 / 已崩溃
- 秘密动机（区别于表面动机）
- 线索载体：此 NPC 能揭示哪条调查线索（可为空）
```

**`agent_types` 合法值：** `npc`、`monster`、`plot`、`lore`、`rules`

---

## 注入层次顺序

Skill 内容在 Agent 执行时按以下顺序注入：

1. **Agent System Prompt**（开发者维护，不可修改）
2. **创作风格**（PromptProfile，规则集级别）
3. **Skill 内容**（工作区级别，本功能）← 当前
4. **任务描述**（运行时用户意图）
5. **知识库检索结果**（RAG）

---

## 与"创作风格"（PromptProfile）的区别

| 维度 | 创作风格（PromptProfile）| Skill |
|-----|------------------------|-------|
| 作用范围 | 规则集级别，影响所有工作区 | 工作区级别 |
| 编写者 | 规则集管理员 | 工作区用户 |
| 用途 | 语气、风格、措辞偏好 | 结构化内容要求、框架约束 |
| 可针对 Agent 类型 | ❌ 全局应用 | ✅ 可指定 agent_types |

---

## 如何添加 Skill

1. 进入工作区设置页（点击顶部工具栏的设置图标，或从工作区列表进入）
2. 找到"Skill"区块
3. 点击"添加 Skill"
4. 填写名称、描述、选择适用 Agent 类型，编写 Skill 内容
5. 点击"保存"

保存后，下次执行 Workflow 时该 Skill 会自动注入到对应 Agent。

## CoC NPC 框架示例（完整）

```
在创作 NPC 时，必须包含以下维度：

**职业**：NPC 在 1920s 社会中扮演的角色（如医生、记者、古董商、警察等）。

**神话接触程度**：
- 无：普通人，对神话一无所知
- 轻微：接触过异常事件，但未深入理解
- 深度：对神话有实质性了解，可能导致理智损失

**心理稳定性**：
- 正常：理智值 50+，行为可预测
- 不稳定：有明显创伤或偏执，行为有时难以预测
- 已崩溃：神话接触导致理智瓦解，行为不可预测

**秘密动机**：与表面动机不同的真实驱动力（不得为空）。

**线索载体**：此 NPC 掌握或能引导到的调查线索（可为空，但若非空则需具体说明）。
```
