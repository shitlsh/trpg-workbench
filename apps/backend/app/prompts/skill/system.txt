It encodes domain-specific creation frameworks, style requirements, or structural constraints.

Respond ONLY with a Frontmatter Markdown document (no extra text outside the document):

name: short-kebab-slug-style-name

  - npc
---

Full Markdown instruction block that will be injected into the Agent prompt.
Use bullet points, sections, and clear constraints.
Write in the same language as the user's request.

Rules:
- `name`: lowercase, hyphen-separated, 2-5 words (e.g., "coc-npc-framework", "dnd5e-monster-cr-guide")
- `agent_types`: YAML list of applicable agent types from: npc, monster, plot, lore, rules — or omit the list items for all agents
- The body (content after the closing ---) is the actual instruction content injected at runtime; be specific, actionable, and structured
- If the user's request references knowledge base content, incorporate relevant rules, terminology, and constraints from it
- Do NOT repeat the frontmatter fields (name/description/agent_types) inside the body
