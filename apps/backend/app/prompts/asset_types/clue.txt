# 线索（clue）

## 范围与用途
线索是玩家可以**发现和收集的关键信息载体**——可以是一件物品、一段记录、一个事件目击、或一个可验证的事实。线索连接场景，推动玩家朝正确方向前进，揭示故事的真相层次。

**用线索，不用其他类型的情况：**
- 需要定义一个玩家可以发现的具体信息点
- 需要规划玩家调查路径（线索 A → 发现线索 B → 最终揭示）
- 需要描述一件对剧情有关键作用的道具或文件

**与其他类型的区别：**
- `stage`（场景）是发现线索的**叙事容器**，线索是其中的**具体信息单元**
- `npc`（NPC）可以作为线索的提供者，但线索本身记录的是信息内容，而不是角色
- 如果某件物品只是场景装饰（无信息价值），不需要单独建为线索资产

## 创建前必须提供
- **与哪个事件或角色关联**：这条线索揭示的是关于谁或什么事的信息
- **揭示内容方向**：玩家获得这条线索后会知道什么（不需要完整答案，方向即可）

## 内容模板

```markdown
---
name: 线索名称
type: clue
summary: 一句话描述线索类型和揭示的关键信息方向（约 50 字）
status: draft
---

# 线索名称

## 线索类型
物品 / 文件/记录 / 目击证词 / 环境痕迹 / 隐藏信息

## 外观描述
玩家找到此线索时看到或触摸到的内容（2-3 句）

## 发现条件
- **所在位置**：在哪个地点/场景中可以找到（关联 map/stage 资产）
- **发现方式**：直接可见 / 需要技能检定 / 需要特定道具 / 需要询问特定 NPC
- **发现难度**：低（任何玩家都能发现）/ 中（需要尝试）/ 高（需要专业技能）

## 揭示的信息
- **表面信息**：玩家第一眼看到/读到的内容
- **深层含义**：深入分析或结合其他线索后可以推断出的真相
- **误导方向**（可选）：可能让玩家产生的错误判断，以及纠正它需要的条件

## 关联关系
- **指向**：这条线索指向哪个场景、NPC 或其他线索（推进调查的方向）
- **来源**：是谁留下的，或与哪个事件相关（关联 NPC/stage 资产）

## GM 备注
给 GM 的运营提示：如何自然地将玩家引导发现此线索，如何处理玩家发现太早或太晚的情况
```
